# 功能需求文档

## 概述

本文档定义了数据库监控系统的功能需求。

## 项目背景

- **项目名称**: 数据库监控系统
- **项目描述**: 自研数据库监控系统，分为Java服务端和Go客户端
- **目的**: 提供主机资源监控能力，包括CPU、内存、磁盘、进程等指标的采集和存储
- **目标用户**: 运维人员、系统管理员

## 项目范围

### 包含范围
- Java服务端支持客户端注册，记录客户端IP和名称
- 服务端根据配置文件向客户端下发监控配置
- Go客户端采集主机资源指标（CPU、内存、磁盘、进程等）
- 客户端定时上报监控数据（间隔由服务端配置）
- 服务端将监控数据存储到SQLite数据库
- 服务端提供监控数据查询API

### 不包含范围
- ~~Web UI界面~~（已调整为HTML5前端整合到后端）
- ~~分布式部署~~
- ~~告警功能~~

## 功能需求列表

### FR-1: 客户端注册功能
- **需求描述**: Java服务端提供客户端注册API，接收客户端IP和名称
- **输入**: 客户端名称(clientName)、主机IP(hostIp)、主机名(hostName)、操作系统类型(osType)
- **输出**: 客户端ID(clientId)、监控配置(监控项列表、上报间隔)
- **优先级**: 高

### FR-2: 监控配置下发
- **需求描述**: 服务端根据配置文件向客户端下发监控配置（监控项、上报间隔）
- **配置项**: 默认监控项列表、上报间隔(秒)
- **优先级**: 高

### FR-3: CPU监控
- **需求描述**: Go客户端采集主机CPU使用率
- **指标**: CPU使用率百分比(0-100)
- **优先级**: 高

### FR-4: 内存监控
- **需求描述**: Go客户端采集主机内存使用情况
- **指标**:
  - 物理内存使用量(字节)
  - 物理内存总量(字节)
  - 虚拟内存使用量(字节)
  - 虚拟内存总量(字节)
- **优先级**: 高

### FR-5: 磁盘监控
- **需求描述**: Go客户端采集主机磁盘使用情况
- **指标**:
  - 磁盘使用率百分比(0-100)
  - 磁盘总量(字节)
  - 磁盘使用量(字节)
- **优先级**: 高

### FR-6: 进程监控
- **需求描述**: Go客户端采集主机进程信息
- **指标**:
  - 进程数量
  - 进程信息列表(PID、名称、CPU、内存占用)
- **优先级**: 中

### FR-7: 监控数据上报
- **需求描述**: Go客户端定时上报监控数据到服务端
- **上报间隔**: 可配置，默认30秒
- **协议**: HTTP POST
- **优先级**: 高

### FR-8: 监控数据存储
- **需求描述**: 服务端接收监控数据并存储到SQLite数据库
- **存储内容**: 所有监控指标、上报时间戳
- **优先级**: 高

### FR-9: 监控数据查询
- **需求描述**: 服务端提供监控数据查询API
- **功能**:
  - 按客户端ID查询历史数据
  - 查询最新数据
  - 按时间范围查询
- **优先级**: 中

### FR-10: 前端界面
- **需求描述**: 提供HTML5前端界面展示监控数据
- **技术栈**: HTML5 + CSS3 + JavaScript
- **部署方式**: 整合到Java后端项目中
- **优先级**: 中

## 业务流程

```
┌─────────┐    注册请求     ┌─────────┐
│  Go客户端 │ ───────────> │ Java服务端│
└─────────┘               └────┬────┘
     │                          │
     │  注册响应(配置)           │ 保存客户端信息
     │ <─────────────────────────│
     │                          │
     ▼                          ▼
定时采集监控数据          ┌─────────┐
     │                   │ SQLite  │
     │ 上报监控数据       │  数据库  │
     └─────────┐         └────┬────┘
               └───────────>│
                             │
                        存储监控数据
```

## 验收标准

### AC-1: 客户端注册
- **Given**: 服务端已启动，监听指定端口
- **When**: 客户端发送注册请求（包含IP和名称）
- **Then**: 服务端返回成功响应，包含监控配置（监控项列表、上报间隔）

### AC-2: 监控数据上报
- **Given**: 客户端已注册成功，收到监控配置
- **When**: 达到上报间隔时间
- **Then**: 客户端采集监控数据并发送到服务端

### AC-3: 监控数据存储
- **Given**: 服务端收到客户端上报的监控数据
- **When**: 数据格式正确
- **Then**: 服务端将数据存储到SQLite数据库

### AC-4: CPU监控
- **Given**: Go客户端运行中
- **When**: 执行监控采集
- **Then**: 正确获取CPU使用率百分比

### AC-5: 内存监控
- **Given**: Go客户端运行中
- **When**: 执行监控采集
- **Then**: 正确获取物理内存和虚拟内存使用情况

### AC-6: 磁盘监控
- **Given**: Go客户端运行中
- **When**: 执行监控采集
- **Then**: 正确获取磁盘占用率

### AC-7: 进程监控
- **Given**: Go客户端运行中
- **When**: 执行监控采集
- **Then**: 正确获取指定进程的PID和资源占用

### AC-8: 前端展示
- **Given**: 服务端运行中
- **When**: 用户访问前端页面
- **Then**: 显示客户端列表和监控数据
