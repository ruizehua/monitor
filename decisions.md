# 架构决策记录

## 概述

本文档记录了项目的关键架构决策。

## ADR-001: 技术栈选择

**状态**: 已接受

**背景**: 需要选择合适的技术栈构建监控系统。

**决策**:
- Java 17 + Spring Boot 3.4.11 作为服务端
- Go 1.21+ 作为客户端
- SQLite 作为数据库

**理由**:
- Java生态成熟，适合企业级后端服务
- Go性能好，适合客户端采集任务
- SQLite轻量级，无需额外部署

## ADR-002: 架构风格

**状态**: 已接受

**背景**: 需要确定系统架构风格。

**决策**: 采用客户端-服务端架构。

**理由**:
- 清晰的职责分离
- 易于扩展和维护
- 支持多客户端接入

## ADR-003: 数据库选型

**状态**: 已接受

**背景**: 需要选择合适的数据库。

**决策**: 使用SQLite作为数据库。

**理由**:
- 轻量级，无需独立服务
- 文件级存储，便于备份
- 适合单机部署场景

## ADR-004: API设计风格

**状态**: 已接受

**背景**: 需要确定API设计风格。

**决策**: RESTful风格。

**理由**:
- 标准化接口设计
- 易于客户端集成
- 良好的可扩展性
