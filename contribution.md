# 贡献指南

## 概述

本文档定义了项目的开发约定和贡献流程。

## 分支策略

| 分支类型 | 命名规则 | 用途 |
|----------|----------|------|
| main | main | 稳定版本分支 |
| develop | develop | 开发分支 |
| feature | feature/xxx | 功能特性分支 |
| bugfix | bugfix/xxx | Bug修复分支 |

## 提交信息规范

```
<type>: <description>

[body]

[footer]
```

### 类型说明

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug修复 |
| docs | 文档更新 |
| style | 代码格式调整 |
| refactor | 代码重构 |
| test | 测试用例 |
| chore | 构建/工具更新 |

## PR流程

1. 创建特性分支
2. 提交代码
3. 创建PR
4. 指定审查者
5. 通过审查后合并

## 代码评审准则

- 代码符合编码规范
- 有足够的测试覆盖率
- 没有潜在的性能问题
- 没有安全漏洞
